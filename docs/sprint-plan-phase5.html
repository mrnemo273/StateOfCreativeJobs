Sprint Thesis

Credibility Before Distribution

The backlog has two natural clusters: features that make the index more trustworthy (methodology, confidence, footnotes, benchmarks, skills analysis) and features that distribute it further (PDF reports, email digests, surveys, emerging roles).

Building distribution before credibility is a mistake. If someone gets your PDF report and the first thing they notice is unsourced data, no methodology link, and no confidence indicators — the report hurts more than it helps. The credibility layer needs to exist before any content leaves the site.

Phase 5 ships the entire credibility cluster. Phase 6 then ships distribution (PDF, email, editorial) on top of a research-grade foundation. Phase 7 tackles the ambitious community features (survey, emerging roles).

Build order recommendation: E14 → G1 → F3 → A3 → E16 (methodology first, benchmarks last)
Execution Sequence
Suggested Build Order

Each step builds on what came before. The methodology page goes first because everything links to it.

Step 1

E14 — Methodology Page

Create /methodology route, server component, static content. This becomes the canonical reference URL that G1 and F3 both link to. Ship it first so subsequent features can reference a real page.

Step 2

G1 — Confidence Indicators

Create ConfidenceBadge component and confidenceMap.ts. Modify 8 section components to add badges to headers. Tooltips link to /methodology. This is the widest-touching change — get it in early to stabilize section headers.

Step 3

F3 — Footnotes & Annotations

Create DataFootnote and ChartAnnotation components. Add automatic freshness notes and curated event markers. These layer onto the same sections that now have confidence badges — building on the same header pattern.

Step 4

A3 — Skills Gap Analyzer

Create SkillsGapSection component, embed in role deep-dive between Skills Signal and Posting Analysis. Self-contained — no cross-references to other sprint features. Clean standalone build.

Step 5

E16 — Industry Benchmarks

Create /benchmarks route, benchmarks.ts aggregation logic, BenchmarksPage component. References /methodology in footer. The most self-contained feature in the sprint — new page, new logic, minimal interaction with existing components.
